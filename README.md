## 📌 Project Overview
This project demonstrates an end-to-end data pipeline that simulates IoT logistics data (Temperature, Humidity, GPS), securely stores the records on a local Ethereum blockchain, retrieves the data via smart contract calls, and processes it for final visualization. 

## 🛠️ Technologies Used
* **Blockchain:** Ganache (Local RPC Provider), Web3.py, Solidity, Remix IDE
* **Data Processing:** Python, Pandas, NumPy
* **Data Visualization:** Matplotlib, Seaborn
* **Environment:** Visual Studio Code (VS Code)

## 📂 Repository Structure
* `IoTDataStorage.sol` - The Solidity smart contract deployed to Ganache to store and manage the IoT records.
* `upload_data.py` - Connects to the Ganache provider and pushes 100 simulated IoT records from a local CSV to the smart contract.
* `check_blockchain_records.py` - Interacts with the blockchain to verify the total number of stored records and fetches a sample tuple to confirm data integrity.
* `preprocessing.py` - Retrieves all 100 records directly from the blockchain, cleans the string data (extracting pure numeric values from formats like "2.55°C" and "42%"), and exports the sanitized dataset.
* `cleaned_iot_data.csv` - The final, sanitized dataset generated by the preprocessing script.
* `visualization.py` - Reads the cleaned CSV file and generates an analytical line plot tracking sensor readings over time.
* `iot_sensor_plot.png` - The final visual output graph.

## 🚀 Step-by-Step Execution Guide

### 1. Blockchain Setup
1. Launch **Ganache** and create a quickstart workspace (RPC Server: `HTTP://127.0.0.1:7545`).
2. Open **Remix IDE** and compile `IoTDataStorage.sol` using the **Paris** EVM version.
3. Deploy the contract using the **Dev - Ganache Provider** environment.
4. Copy the newly generated Contract Address.

### 2. Configure Python Scripts
Paste your deployed Contract Address into the `contract_address` variable inside the following scripts:
* `upload_data.py`
* `check_blockchain_records.py`
* `preprocessing.py`

### 3. Run the Pipeline
Execute the scripts in the terminal in the following order:

**Step A: Push Data to Blockchain**
```powershell
python upload_data.py
